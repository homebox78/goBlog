### 핵심 진단
**"남의 글(경쟁 블로그) 학습 루프"는 돌고 있지만, "내 글 성과 루프"는 존재하지 않는다.**
발행하면 거기서 끝. 노출수·클릭률·체류시간·수익이 하나도 안 들어온다.
지금 대시보드 숫자(글 수·발행 성공·품질점수·AI 비용)는 **내부 생산 지표**지 독자 반응이 아니다.

## A. 성과 데이터 수집 — **최우선** (이게 없으면 B·C는 의미 없음)

- [ ] **A-0. 선행 확인**: Search Console에 각 발행처가 **사이트 등록 + 소유확인** 되어 있는지
  - [ ] 워드프레스 `hom2box.com/wordpress`
  - [ ] 티스토리 (RSS/메타태그 소유확인)
  - [ ] Blogger (자동 연결됨)
  - [ ] 네이버 블로그 → **GSC 대상 아님**. 네이버 서치어드바이저 / 네이버 애널리틱스 별도 필요
- [ ] **A-1. OAuth 스코프 확장** — Blogger에 쓰는 구글 계정 그대로 재사용 (신규 키 발급 불필요, 무료)
  - `webmasters.readonly` (Search Console)
  - AdSense는 별도 스코프 + 계정 승인 상태 확인
- [ ] **A-2. 수집기 구현** `apps/api/src/modules/analytics/collectors/`
  - GSC: 일자별 노출수·클릭수·CTR·평균순위·**유입 쿼리** (page 단위)
  - AdSense: 일자별 수익·RPM (page 단위)
  - `publish_jobs.publishedUrl` ↔ GSC page URL 매칭으로 **글 단위 성과** 연결
- [ ] **A-3. 저장** — 기존 `AnalyticsDaily` 모델 사용 (스키마 이미 존재)
  - 부족한 필드 확인: 체류시간·이탈률은 GSC에 없음 → GA4 필요 여부 판단
- [ ] **A-4. 스케줄** — 하루 1회 (GSC 데이터는 2~3일 지연됨을 감안)
- [ ] **A-5. 대시보드** — 글별 노출/클릭/순위/수익 표시 + 상위·하위 글 랭킹

**비용: 무료** (GSC/AdSense API 무료). 리스크: GSC 데이터 지연·샘플링, 네이버는 별도 경로.

## 개발 시 주의 (실제로 사고 났던 것들)

- **이미지 생성은 배포된 서버에서만** — 로컬 개발 API로 만들면 `MEDIA_PUBLIC_URL`이 없어 죽은 링크가
  공유 운영 DB에 박힌다. (지금은 DB 가드가 막지만, 애초에 서버에서 하는 게 맞다)
- **이미지 재생성 후에는 새로고침** 하고 편집할 것. 오래된 화면에서 저장하면 본문이 되돌아간다
  (지금은 409로 막힘).
- **배포는 작업 중이 아닐 때** — 우아한 종료를 넣어 진행 중 요청은 살아남지만, 굳이 겹칠 필요 없다.
